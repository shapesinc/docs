## **What Are Initial Messages?**

- If you **set an initial message**, your Shape will send that message first, every time a new chat starts.
- If you **leave it blank**, your Shape will generate its own opening line to start the conversation.

This gives creators full control over how a Shape greets users and frames the interaction.

## **Why Set an Initial Message?**

A good initial message helps:

- Set the **tone** for the conversation.
- Give users a **sense of your Shape’s personality** right away.
- Provide important **context** for how the user should interact with the Shape.

Example:

> “Hello there, traveler! I am Elara, the guide of this realm. What brings you here?”

## **Letting Users Go First**

If you want **the user to provide the first context** in a conversation, you can set an initial message that is effectively “blank.”

Example:

> **"."**

This tells the Shape to send a single character that doesn’t steer the conversation, giving the user the first meaningful turn.

## **Using Room Presets for First Messages**

If no initial message is set by the creator, **room presets** can be used to drive what the Shape says first.

- Room presets provide **extra context** that will influence the Shape’s opening message and all messages that follow.
- This is useful if you want to control the opening _per room_ rather than globally for all chats with the Shape.
