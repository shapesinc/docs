## **What Are Initial Messages?**

- If you **set an initial message**, your Shape will send that message first, every time a new chat starts. This only works with free will enabled.
- If you **leave it blank**, your Shape will generate its own opening line to start the conversation.

This gives creators full control over how a Shape greets users and frames the interaction.

## **Why Set an Initial Message?**

A good initial message helps:

- Set the **tone** for the conversation.
- Give users a **sense of your Shape’s personality** right away.
- Provide important **context** for how the user should interact with the Shape.

Example:

> “Hello there, traveler! I am Elara, the guide of this realm. What brings you here?”

## **Letting Users Go First**

If you want **the user to provide the first context** in a conversation, you can leave the field blank, it is completely optional.

## **Using Custom Chat Instructions for First Messages**

If no initial message is set by the creator, [**custom chat instructions**](/custominstructions-chats) can be used to drive what the Shape says first.

- Custom chat instructions provide **extra context** that will influence the Shape’s opening message and all messages that follow.
- This is useful if you want to control the opening _per chat_ rather than globally for all chats with the Shape.

## Free Will

- If free will is enabled, the Shape will always go first, whether an initial message is set or not.
- If free will is disabled, the user will always go first, whether an initial message is set or not.
- Free Will is controlled inside the chat's **AI Configurations** settings.
