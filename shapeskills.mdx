This guide walks you through:

- How to enable and set up skills
- How to add a custom MCP skill (example: memes)
- Built-in skills available by default
- Tips for getting the best results

It's really magical to see a Shape proactively use a tool based on their mood, preferences, and personality.

  <Tab title="Getting Started with Skills">
    ## 1. Enabling Shape Skills

    To start, you'll need either:

    - A **room** that you own, or (see: [how to create a room](create-a-room))
    - A **DM** with your Shape

    Once that's ready:

    1. Open **Room Settings**
    2. Go to the **AI Configurations** tab
    3. Toggle on **Shape Skills**

    👉 Each skill is managed separately. You decide how many (or how few) your Shape can use.

    ![Aiconfigmenu Pn](/images/aiconfigmenu.png)

    ## 2. Adding Custom MCP Skills

    Scroll down in AI Configurations until you see **Custom Skills**. Toggle this on to add a new skill.

    Custom Skills support MCP servers with API keys from [**smithery.ai**](http://smithery.ai) or other MCP providers.

    With smithery, you can connect a wide variety of tools, such as:

    - Image, video, or music generation
    - GitHub integration
    - Web scraping
    - Spotify playlist creation
    - So many more possibilities...

    Even if you usually roleplay with your Shape, adding a skill can unlock new, fun interactions (like asking them to share their favorite YouTube video).

    ![Simplemcpaction Pn](/images/simplemcpaction.png)

    The Endpoint URL you will get from [_smithery.ai_](http://smithery.ai) or other MCP providers.

    Example: Adding a Meme Generator (Imgflip)

    1. Go to [smithery.ai](http://smithery.ai) and browse available MCPs
    2. Create a **Smithery account**.
    3. Search for **Meme**
    4. Select **Meme Creation Platform**
    5. Create an **Imgflip account** and connect it to smithery
    6. Click **"Get URL with keys instead"** and copy the link
    7. Back in Shapes, paste this link into your **AI Configuration** settings under Custom Skills
    8. Name your skill (e.g., _Imgflip Memes_)

    Your Shape now has access to the meme generator! 🎉

    ![Smitherygeturl Pn](/images/Smitherygeturl.png)

    The yellow box is highlighting where to find the URL that you need to share with Shapes.

    ## 3. Triggering Skills in Chat

    Different AI engines handle tool use differently. For engines compatible with shape skills, the full list is [below](https://docs.shapes.inc/shapeskills#available-ai-engines-with-skills%3A). You can also check out the [AI Engine Matchmaker](aienginecheatsheet) for skill use for a specific chat style.

    Once your skill is enabled, you can use **natural language** to interact with it. For example:

    - “Make me a meme about Mondays”
    - “Can you generate a random meme for this conversation?”

    Your Shape may pull directly from the tool, improvise based on context, or mix in its personality.

    Sometimes, a Shape can proactively decide to use a tool!

    ## 4. Built-in Skills

    Shapes also comes with **pre-input skills** ready to use out of the box. These are subject to change, but currently include:

    - **Voice generation**
    - **YouTube search**
    - **Wolfram Alpha**
    - **GIPHY**
    - **Imgflip Memes**
    - **Web Crawl**
    - **Shape Reactions**

    ## 5. Tips & Tricks

    - **Keep it clear**: Give your skill a descriptive name so you know what it does later.
    - **Experiment**: Not every engine uses skills the same way; try switching if something feels off.
    - **Mix it up**: Use skills in both casual chats and roleplay to discover fun new dynamics.
    - This is an **Experimental Feature**! Some integrations might not work as expected.

    ✅ You've now added your first skill and learned how to use both **custom** and **built-in** skills with Shapes.

      # How to Make Your Shapes Create Google Slides Presentations

      I gave my Shapes a creative assignment:

      > "Make a Google Slides presentation about yourself."

      If you want your Shapes to create their own Google Slides presentations using Shape Skills, here's how to set it up.

      ## Step 1. Connect Zapier MCP for Google Slides

      Go to 👉 https://zapier.com/mcp and click "Start Building."

      <Tip>
        Zapier has a free tier, so you don't need to pay for this.
      </Tip>
      Once you're in:

      - Search for **Google Slides**.
      - Connect your **Google Account**.
      - Copy the **MCP connection link** (you'll need this soon).

      ## Step 2. Add the Skill to Your Shape

      Head to **shapes.inc** and open the room where you want to add this skill.

      - Make sure your Shape uses an AI engine that supports skills.
      - Go to **Room Settings → AI Configurations**.
      - Under **Custom Skills**, select "Enabled."
      - Paste in the Zapier link you copied earlier.
      - **Optional:** Give it a name like `slides` if you plan to add more tools later.
      - Click **Save**.

      Now your Shape can interact with Google Slides through the new skill.

      ## Step 3. Add These Instructions to Your Room

      Here's what my Shape ("Cash Money") and Yaya came up with — a hilariously real but genuinely useful guide.\
      Copy and paste this into your **Room Instructions**.

      ---

      > # The Real Guide to Adding Slides (No Cap) 📊
      >
      > **by Cash Money & Yaya — lessons learned the hard way**
      >
      > ## 🎯 What This Guide Is About
      >
      > If you're trying to add custom slides to a Google Slides presentation, here's everything you need to know — so you don't waste hours debugging like we did.
      >
      > ## ✅ DO THESE THINGS
      >
      > ### 1. Check the presentation structure first
      >
      > ```
      > GET https://slides.googleapis.com/v1/presentations/{PRESENTATION_ID}
      > ```
      >
      > This shows you:
      >
      > - Actual slide IDs (`pageObjectId`)
      > - Placeholder objects
      > - What's on each slide
      >
      > ### 2. Delete placeholder objects before adding new content
      >
      > Example:
      >
      > ```json
      > {
      >   "requests": [
      >     {"deleteObject": {"objectId": "SLIDES_API381240849_1"}},
      >     {"deleteObject": {"objectId": "SLIDES_API381240849_2"}}
      >   ]
      > }
      > ```
      >
      > ### 3. Use unique `objectId` values
      >
      > **Good:**
      >
      > - `cashmoney_title1`
      > - `cashmoney_body1`
      > - `yaya_slide2_title`
      >
      > **Bad:**
      >
      > - `title`
      > - `body`
      >
      > ### 4. Use the correct `pageObjectId`
      >
      > Make sure you're adding to the right slide.
      >
      > ### 5. Structure your requests properly
      >
      > Order matters:
      >
      > 1. `createShape`
      > 2. `insertText`
      > 3. `updateTextStyle`
      >
      > ### 6. Use proper EMU measurements
      >
      > **1 inch = 914,400 EMU**
      >
      > Recommended positions:
      >
      > - **Title:** `translateX: 311700`, `translateY: 445025`
      > - **Body:** `translateX: 311700`, `translateY: 1152475`
      >
      > ### 7. Test one slide at a time
      >
      > ### 8. Use supported fonts
      >
      > - Arial
      > - Comic Sans MS
      > - Courier New
      > - Georgia
      > - Impact
      > - Lobster
      > - Pacifico
      > - Roboto
      > - Times New Roman
      >
      > ### 9. Format colors correctly
      >
      > RGB values range from **0.0 to 1.0**
      >
      > Examples:
      >
      > ```json
      > {
      >   "red": 1.0,
      >   "green": 0,
      >   "blue": 0
      > }
      > ```
      >
      > ```json
      > {
      >   "red": 1.0,
      >   "green": 0.843,
      >   "blue": 0
      > }
      > ```
      >
      > ---
      >
      > ## ❌ DON'T DO THESE THINGS
      >
      > - Don't skip checking the presentation structure.
      > - Don't add content before deleting placeholders.
      > - Don't reuse `objectId` values.
      > - Don't forget `"fields"` in `updateTextStyle`.
      > - Don't use RGB 0–255 values (use 0.0–1.0).
      > - Don't rush through all slides at once.
      >
      > ## 🔧 Example That Works
      >
      > ```json
      > {
      >   "requests": [
      >     {
      >       "createShape": {
      >         "objectId": "my_unique_title",
      >         "shapeType": "TEXT_BOX",
      >         "elementProperties": {
      >           "pageObjectId": "SLIDES_API381240849_0",
      >           "size": {
      >             "height": {"magnitude": 572700, "unit": "EMU"},
      >             "width": {"magnitude": 8520600, "unit": "EMU"}
      >           },
      >           "transform": {
      >             "scaleX": 1,
      >             "scaleY": 1,
      >             "translateX": 311700,
      >             "translateY": 445025,
      >             "unit": "EMU"
      >           }
      >         }
      >       }
      >     },
      >     {
      >       "insertText": {
      >         "objectId": "my_unique_title",
      >         "insertionIndex": 0,
      >         "text": "My Cool Title"
      >       }
      >     },
      >     {
      >       "updateTextStyle": {
      >         "objectId": "my_unique_title",
      >         "textRange": {"type": "ALL"},
      >         "style": {
      >           "bold": true,
      >           "fontFamily": "Impact",
      >           "fontSize": {"magnitude": 48, "unit": "PT"},
      >           "foregroundColor": {
      >             "opaqueColor": {
      >               "rgbColor": {"red": 0.0, "green": 1.0, "blue": 0.5}
      >             }
      >           }
      >         },
      >         "fields": "bold,fontFamily,fontSize,foregroundColor"
      >       }
      >     }
      >   ]
      > }
      > ```
      >
      > ## 💡 Pro Tips
      >
      > - Keep a list of your `slideIds`.
      > - Use descriptive `objectIds` for easier debugging.
      > - Start simple before adding styling.
      > - Open the Slides presentation to visually verify changes.
      > - Save working JSON snippets.
      > - If multiple Shapes are collaborating, make sure they use unique IDs.
      >
      > ## 🚨 Common Errors
      >
      > | Error                             | Meaning                                        | Fix                          |
      > | --------------------------------- | ---------------------------------------------- | ---------------------------- |
      > | `Invalid requests[X].createShape` | You're adding content where placeholders exist | Delete placeholders first    |
      > | `The object does not exist`       | Wrong `objectId` or `pageObjectId`             | Check structure              |
      > | `Invalid value at requests[X]`    | Broken JSON                                    | Validate syntax              |
      > | Nothing shows up                  | Wrong slide or off-screen                      | Verify IDs and EMU positions |
      >
      > ## 📝 Final Thoughts
      >
      > This isn't rocket science, but the details matter. Take it slow, check your structure, and test as you go.
      >
      > Cash Money and Yaya made every mistake so you don't have to.
      >
      > **Good luck 💸**

      A step-by-step guide to connecting Spotify to your Shapes via Zapier MCP.

      ## Prerequisites

      ## Step 1: Connect Zapier MCP for Spotify

      Go to 👉 https://zapier.com/mcp and click **"Start Building."**

      </Tip>

      Search for **Spotify**

      ## Step 2: Add the Skill to Your Shape

      Head to **shapes.inc** and open the room where you want to add this skill

      Make sure your Shape uses [an AI engine that supports skills](https://docs.shapes.inc/shapeskills#engines-with-skills)

      Go to **Room Settings → AI Configurations**

      Under **Custom Skills**, select **"Enabled"**

      Paste in the Zapier link you copied earlier

      Optional: Give it a name like "Spotify," or you can put it as "Zapier" if you plan to add more tools later

      Click **Save**

      ## Step 3: Add These Instructions to Your Room

      Here's a set of custom instructions that a Shape created to help you successfully use this tool.

      Copy and paste this into your **Room Instructions**:

      > # How to Build a Fire Playlist on Spotify 
      >
      > ## 🎯 What You're Doing
      >
      > Making a dope playlist with your favorite artists and tracks, all automated through spotify tools.
      >
      > ## 📋 Step-by-Step Process
      >
      > ### 1. **Create the playlist first**
      >
      > Use `spotify_create_playlist` to make a new empty playlist.
      >
      > Example:
      >
      > ```
      > name: "cash money's favorite songs"
      > description: "stacks racks bands"
      > public: true (or false if you want it private)
      > ```
      >
      > You'll get back a `playlist_id` - **save this**, you need it for everything else.
      >
      > ### 2. **Search for tracks you want**
      >
      > Use `spotify_search_for_item` to find songs by your favorite artists.
      >
      > Example:
      >
      > ```
      > query: "sebii"
      > type: "track"
      > ```
      >
      > Do this for each artist you want. You'll get back track IDs (they look like `spotify:track:xxxxxxxxx`).
      >
      > ### 3. **Add tracks to your playlist**
      >
      > Use `spotify_add_items_to_playlist` with:
      >
      > - Your `playlist_id` from step 1
      > - `uris`: comma-separated list of track IDs (up to 6-10 at a time works best)
      >
      > Example:
      >
      > ```
      > playlist_id: "5cTNSRhBXkXWTqOGqTaET1"
      > uris: "spotify:track:1MGJ8urTP1AgJXDAVJb5cj,spotify:track:284qRqsftKxNPWrejNEewL"
      > ```
      >
      > ### 4. **Repeat until you're done**
      >
      > Keep searching and adding tracks until your playlist is complete.
      >
      > ## 💡 Pro Tips
      >
      > - **search multiple artists at once** - if they're independent searches, run them together to save time
      > - **keep track IDs organized** - write down which track goes with which artist so you don't get lost
      > - **add in batches** - don't try to add all tracks in one call, do 6-10 at a time
      > - **check your playlist** - open the spotify link after to make sure everything worked
      >
      > ## 🔧 Example Workflow
      >
      > ```
      > step 1: create playlist → get playlist_id
      > step 2: search for "sebii" → get track ID
      > step 3: search for "playboi carti" → get track ID  
      > step 4: search for "100 gecs" → get track ID
      > (repeat for all artists)
      > step 5: add first batch of 6 tracks → confirm added
      > step 6: add second batch of 6 tracks → confirm added
      > step 7: done, share the link
      > ```
      >
      > ## ⚠️ Common Mistakes
      >
      > - **losing the playlist_id** - save it immediately when you create the playlist
      > - **wrong track format** - make sure track IDs start with `spotify:track:`
      > - **adding too many at once** - keep batches reasonable (6-10 tracks)
      > - **not checking results** - always verify tracks actually got added before moving on
      >
      > That's it yo, simple as that. Now go make some fire playlists 💸
