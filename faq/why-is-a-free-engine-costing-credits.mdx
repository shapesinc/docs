---
title: "Why is a free engine costing credits?"
description: "Check engine settings when credits are used unexpectedly."
---

Free engines do not cost credits.

Credits are only used when you trigger a response from a **premium engine**.

If credits are being used unexpectedly, a premium engine is probably being used instead of a free one. Check these settings:

1. **Shape Settings → AI Engine**\
   From **Home**, go to **Your Shapes** and open the Shape. On the Shape page, open the sidebar menu and tap **AI Engine** under **Shape Settings**.
   - If you are overriding the Shape's default engine, make sure the selected engine is a **free engine**.
   - If you want to use the Shape's primary engine, the override toggle should be **off**.
   - If you are using a primary AI Engine, choose a free engine. Free engines do not show a credit warning below the selection.
2. **AI Configurations → Prevent engine override**\
   Check whether **Prevent engine override** is enabled in your chat. Open the chat title to find this setting.
   - Turn this **on** if you do _not_ want your primary engine to be overridden.
   - Turn this **off** if you _do_ want to allow engine overrides.
3. **Chat Settings → Personal AI Preferences**\
   Open the chat title and confirm that your **personal AI preference** is set to **Do not use my credits**.
   - There is an **auto-upgrade** feature that will automatically switch your chat to a premium engine if credits are available.
   - Make sure this setting is enabled, then reload the app and check it again.
   - This setting is user-specific, and each user will need to set this in a chat. The chat owner has no impact on this setting.

Some actions and premium engines may still require credits. Prices, available engines, and credit rules can change, so check the credit notice shown in chat before sending if you are trying to avoid credit usage.

If this still looks wrong, ask in the [support chat](https://talk.shapes.inc/support).

Be ready to invite a staff member to the chat and make them an **owner** for a short time so we can check the settings.
